Project Overview

User Story A - User Authentication

User Authentication is important because it ensures that each user has their own personalized account, allowing them to save favorites or keep track of how they have used the application.

User Story B - Restaurant Search

Restaurant Search gives users flexibility in how they find restaurants, optimizing options for the user.

User Story C - Geolocation

Geolocation utilizes Google Maps to show precise locations to users, allowing them to easily view which restaurants are near them.

User Story D - Restaurant Details

Restaurant Details is important because it allows users to get a more in-depth view of restaurants they may be interested in and make more informed decisions.

User Story E - Favorites

Favorites is important because it allows users to easily keep track of their favorite restaurants and access their details quickly.

User Story F - Extra Credit (Writing Reviews/Setting Passwords)

This is important because both extra goals add an extra layer of interactivity and convenience for the user, incentivizing them to continue using the platform.

User Story 2 - User Authentication

User Authentication was essential because each SpotifyWrapped account is linked to the user's unique Spotify account, ensuring that the wraps our app generated were personalized and accurate. It also allowed these wraps to exist after the user logs out.

Custom User Story 6 - Custom Timeframes

Our project empowers users to generate personalized Spotify Wrapped summaries based on custom timeframes, such as all-time, last year, or last month, without relying on external packages like Spotipy.

User Story 3 - Responsive UI

Our app features an amazingly responsive user interface, so that users are able to access SpotifyWrapped on any device, whether it be a desktop, tablet, or mobile phone.

User Story 4 - Data Privacy

Our project emphasizes secure handling of user data by using Spotify's API responsibly, ensuring that data remains private and never leaves the app’s ecosystem. We utilized our .gitignore and .env files to ensure that our Spotify API keys were never exposed to the public.

Custom User Story 17 - Wireframe and UI Flow

To ensure the app's user experience was smooth, we created high-fidelity mockups using Figma. These wireframes helped visualize the UI flow, including key screens like the home page, playlist generation, and user settings, which directly influenced the app's navigation design.

Custom User Story 15 - Contact the Developers Page

We created a Contact the Developers page that allows users to submit their feedback, report bugs, or ask questions. This page includes a simple form and ensures that users can easily get in touch with the development team for any issues or suggestions.

Custom User Story 5 - Play Song Clips

To enhance the user experience, we implemented the ability to play short song previews directly from the Wrapped summary. This feature allows users to listen to their favorite tracks and relive their top moments from the past year. Unfortunately, recent changes to Spotify's API have made this feature inaccesible.

Custom User Story 14 - Transitions and Hover Effects

We added smooth transitions and hover effects to our app’s elements to improve interactivity and create a visually appealing experience. These effects make the app feel more dynamic, especially when navigating through user playlists or viewing song details.

Custom User Story 10 - Dark Mode

Recognizing the preference of many users for dark mode, we introduced this feature to our app. Users can now toggle between light and dark modes, ensuring a more comfortable viewing experience, especially in low-light environments.
